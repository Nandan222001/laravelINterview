Section 1: AWS Lambda & Serverless Computing (Questions 1-150)

Event Processing & Integrations (1-30)

Question 1
Explain how AWS Lambda processes events from different event sources (S3, DynamoDB, SQS, API Gateway).
Lambda processes events through different integration patterns:
  • S3: Push-based, async invocation when objects are created/deleted
  • DynamoDB Streams: Poll-based, Lambda polls the stream for changes
  • SQS: Poll-based, Lambda polls queue and processes in batches
  • API Gateway: Synchronous invocation, waits for response
Each source has different retry behaviors and error handling mechanisms.
Question 2
What is the Lambda event object structure for API Gateway proxy integration?
The event object contains: httpMethod, path, pathParameters, queryStringParameters, headers, body (JSON string), requestContext (with request ID, account, and authorizer data), and isBase64Encoded flag.

import json
import boto3
import os
from PIL import Image
from io import BytesIO

s3_client = boto3.client('s3')

def lambda_handler(event, context):
    for record in event['Records']:
        bucket = record['s3']['bucket']['name']
        key = record['s3']['object']['key']
        
        if 'thumbnails/' in key:
            continue
        
        try:
            # Download image from S3
            response = s3_client.get_object(Bucket=bucket, Key=key)
            image_data = response['Body'].read()
            
            # Generate thumbnail
            img = Image.open(BytesIO(image_data))
            img.thumbnail((200, 200))
            
            # Save thumbnail to buffer
            buffer = BytesIO()
            img.save(buffer, format=img.format)
            buffer.seek(0)
            
            # Upload thumbnail to S3
            thumbnail_key = f"thumbnails/{key}"
            s3_client.put_object(
                Bucket=bucket,
                Key=thumbnail_key,
                Body=buffer,
                ContentType=response['ContentType']
            )
            
            print(f"Generated thumbnail: {thumbnail_key}")
            
        except Exception as e:
            print(f"Error processing {key}: {str(e)}")
            raise
    
    return {
        'statusCode': 200,
        'body': json.dumps('Thumbnails generated successfully')
    }
Question 4

Lambda automatically scales to handle concurrent requests. Default account limit is 1000 concurrent executions (can be increased). Reserved concurrency allocates specific capacity to a function. Burst concurrency: initial 3000 (1000 in some regions), then 500/minute increase.
Question 5
Explain the difference between synchronous and asynchronous Lambda invocations.
Synchronous: Caller waits for response (API Gateway, SDK invoke). Errors returned to caller immediately.
Asynchronous: Event queued, Lambda returns 202 (S3, SNS). Automatic retries (2x), can configure DLQ for failures.

Attention Mechanism (151-190)

Question 151
Explain the core concept of attention mechanism in neural networks.
Attention allows the model to focus on relevant parts of the input when processing each element. Instead of compressing all information into a fixed-size vector, attention computes weighted sums of input representations, where weights indicate relevance. This solves the bottleneck problem in seq2seq models.
Question 152
What is the mathematical formula for scaled dot-product attention?
Attention(Q, K, V) = softmax(QK^T / √d_k)V
Where Q=queries, K=keys, V=values, d_k=dimension of keys.
The scaling by √d_k prevents dot products from growing too large, which would push softmax into regions with small gradients.
Question 153
Write Python code to implement basic attention mechanism from scratch using NumPy.
Python - Attention from Scratch attention.py 
import numpy as np

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Scaled Dot-Product Attention
    
    Args:
        Q: Queries (batch, seq_len, d_k)
        K: Keys (batch, seq_len, d_k)
        V: Values (batch, seq_len, d_v)
        mask: Optional mask
    """
    d_k = Q.shape[-1]
    
    # Compute attention scores: Q * K^T / sqrt(d_k)
    scores = np.matmul(Q, K.transpose(0, 2, 1)) / np.sqrt(d_k)
    
    # Apply mask if provided
    if mask is not None:
        scores = np.where(mask == 0, -1e9, scores)
    
    # Apply softmax
    attention_weights = np.exp(scores - np.max(scores, axis=-1, keepdims=True))
    attention_weights = attention_weights / np.sum(attention_weights, axis=-1, keepdims=True)
    
    # Weighted sum of values
    output = np.matmul(attention_weights, V)
    
    return output, attention_weights

# Example usage
batch_size, seq_len, d_k = 2, 4, 8
Q = np.random.randn(batch_size, seq_len, d_k)
K = np.random.randn(batch_size, seq_len, d_k)
V = np.random.randn(batch_size, seq_len, d_k)

output, weights = scaled_dot_product_attention(Q, K, V)
print(f"Output shape: {output.shape}")
print(f"Attention weights shape: {weights.shape}")
Question 169
Write code to implement multi-head attention from scratch.
Python - Multi-Head Attention multihead_attention.py 
import torch
import torch.nn as nn
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8, dropout=0.1):
        super().__init__()
        
        assert d_model % num_heads == 0
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads
        
        # Linear projections
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        
        self.dropout = nn.Dropout(dropout)
        
    def split_heads(self, x, batch_size):
        x = x.view(batch_size, -1, self.num_heads, self.d_k)
        return x.transpose(1, 2)
    
    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)
        
        # Linear projections
        Q = self.W_q(query)
        K = self.W_k(key)
        V = self.W_v(value)
        
        # Split into multiple heads
        Q = self.split_heads(Q, batch_size)
        K = self.split_heads(K, batch_size)
        V = self.split_heads(V, batch_size)
        
        # Scaled dot-product attention
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attn_weights = torch.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)
        
        attn_output = torch.matmul(attn_weights, V)
        
        # Concatenate heads
        attn_output = attn_output.transpose(1, 2).contiguous()
        attn_output = attn_output.view(batch_size, -1, self.d_model)
        
        # Final linear projection
        output = self.W_o(attn_output)
        
        return output, attn_weights

Section 4: RAG (Retrieval-Augmented Generation) (Questions 451-600)

RAG Architecture & Concepts (451-480)

Question 451
Explain what RAG (Retrieval-Augmented Generation) is and why it's important.
RAG combines retrieval from a knowledge base with LLM generation. Instead of relying solely on the model's parametric knowledge, RAG retrieves relevant documents and provides them as context to the LLM. This: (1) Reduces hallucinations, (2) Enables access to up-to-date information, (3) Allows domain-specific knowledge without fine-tuning, (4) Provides source attribution.

Question 461
Write code to implement a RAG system with document chunking.
Python - RAG with Smart Chunking rag_chunking.py 
import openai
import pinecone
from typing import List, Dict

class RAGPipeline:
    def __init__(self, index_name: str):
        self.index_name = index_name
        self.chunk_size = 1000
        self.chunk_overlap = 200
        
        # Initialize Pinecone
        if index_name not in pinecone.list_indexes():
            pinecone.create_index(
                name=index_name,
                dimension=1536,
                metric='cosine'
            )
        
        self.index = pinecone.Index(index_name)
    
    def chunk_text(self, text: str) -> List[str]:
        """Split text into overlapping chunks"""
        chunks = []
        start = 0
        
        while start < len(text):
            end = start + self.chunk_size
            chunk = text[start:end]
            
            # Try to break at sentence boundary
            if end < len(text):
                last_period = chunk.rfind('.')
                if last_period > self.chunk_size * 0.5:
                    chunk = chunk[:last_period + 1]
                    end = start + last_period + 1
            
            chunks.append(chunk)
            start = end - self.chunk_overlap
        
        return chunks
    
    def ingest_documents(self, documents: List[Dict]):
        """Ingest documents with chunking and embedding"""
        all_chunks = []
        
        for doc in documents:
            chunks = self.chunk_text(doc['text'])
            
            for i, chunk in enumerate(chunks):
                all_chunks.append({
                    'id': f"{doc['id']}_chunk_{i}",
                    'text': chunk,
                    'metadata': {
                        'source': doc['id'],
                        'chunk_index': i,
                        **doc.get('metadata', {})
                    }
                })
        
        # Generate embeddings in batches
        self._embed_and_upsert(all_chunks)
    
    def _embed_and_upsert(self, chunks: List[Dict], batch_size: int = 100):
        for i in range(0, len(chunks), batch_size):
            batch = chunks[i:i + batch_size]
            texts = [c['text'] for c in batch]
            
            # Generate embeddings
            response = openai.Embedding.create(
                input=texts,
                model="text-embedding-ada-002"
            )
            
            # Prepare vectors
            vectors = []
            for chunk, emb_data in zip(batch, response['data']):
                vectors.append({
                    'id': chunk['id'],
                    'values': emb_data['embedding'],
                    'metadata': {
                        'text': chunk['text'],
                        **chunk['metadata']
                    }
                })
            
            self.index.upsert(vectors=vectors)
    
    def query(self, question: str, top_k: int = 5) -> str:
        """Query with RAG"""
        # Generate query embedding
        query_emb = openai.Embedding.create(
            input=[question],
            model="text-embedding-ada-002"
        )['data'][0]['embedding']
        
        # Search
        results = self.index.query(
            vector=query_emb,
            top_k=top_k,
            include_metadata=True
        )
        
        # Build context
        context = "\n\n".join([
            match['metadata']['text'] 
            for match in results['matches']
        ])
        
        # Generate answer
        response = openai.ChatCompletion.create(
            model="gpt-4",
            messages=[
                {"role": "system", "content": "Answer based on context."},
                {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"}
            ]
        )
        
        return response['choices'][0]['message']['content']

Pinecone (481-505)

Question 481
What is Pinecone and how does it differ from traditional databases?
Pinecone is a managed vector database designed for similarity search. Unlike traditional databases that use exact matching, Pinecone stores high-dimensional vectors and performs approximate nearest neighbor (ANN) search. It's optimized for embedding-based retrieval with features like namespaces, metadata filtering, and horizontal scaling.

MCP (Model Context Protocol) (601-640)

Question 601
What is MCP (Model Context Protocol) and why was it developed?
MCP is a protocol for secure context sharing between AI systems and data sources. Developed to standardize how LLMs access external data, it provides: (1) Authentication/authorization, (2) Context isolation, (3) Audit logging, (4) Rate limiting, (5) PII protection. It enables secure enterprise AI applications.

Fine-tuning Strategies (641-670)

Question 641
Explain the difference between fine-tuning and prompt engineering.
Prompt Engineering: Craft inputs to guide model behavior. No training required, fast iteration, works with any model. Limited by context window.
Fine-tuning: Train model on custom data to specialize. Requires labeled data and compute, improves performance on specific tasks, changes model weights permanently. Use fine-tuning for consistent behavior, prompt engineering for flexibility.

    expected = hmac.new(
        WEBHOOK_SECRET.encode('utf-8'),

Question 114
Create a Lambda function for processing Stripe webhook events with signature validation.
Python - Stripe Webhook Handler stripe_webhook.py 
import stripe
import os
import json

stripe.api_key = os.environ['STRIPE_SECRET_KEY']
WEBHOOK_SECRET = os.environ['STRIPE_WEBHOOK_SECRET']

def lambda_handler(event, context):
    sig_header = event['headers'].get('stripe-signature', '')
    body = event['body']
    
    try:
        # Verify webhook signature
        stripe_event = stripe.Webhook.construct_event(
            body, sig_header, WEBHOOK_SECRET
        )
    except stripe.error.SignatureVerificationError as e:
        return {
            'statusCode': 400,
            'body': json.dumps({'error': 'Invalid signature'})
        }
    
    # Handle event
    event_type = stripe_event['type']
    
    if event_type == 'checkout.session.completed':
        handle_checkout_completed(stripe_event)
    elif event_type == 'payment_intent.succeeded':
        handle_payment_succeeded(stripe_event)
    elif event_type == 'customer.subscription.created':
        handle_subscription_created(stripe_event)
    
    return {
        'statusCode': 200,
        'body': json.dumps({'received': True})
    }

def handle_checkout_completed(event):
    session = event['data']['object']
    customer_id = session.get('customer')
    amount = session['amount_total'] / 100
    print(f"Checkout completed: ${amount}")

def handle_payment_succeeded(event):
    payment_intent = event['data']['object']
    amount = payment_intent['amount'] / 100
    print(f"Payment succeeded: ${amount}")

def handle_subscription_created(event):
    subscription = event['data']['object']
    print(f"Subscription created: {subscription['id']}")
