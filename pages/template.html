Example Question: How do you implement API rate limiting in Laravel?

Intermediate Security API

Explain how to implement rate limiting for APIs in Laravel to prevent abuse and ensure fair usage.

Answer:

Laravel provides built-in rate limiting through the throttle middleware. You can apply it to routes or route groups to limit the number of requests a user can make within a given time period.

// In routes/api.php
Route::middleware('throttle:60,1')->group(function () {
    Route::get('/user', function () {
        return auth()->user();
    });
});

This limits requests to 60 per minute. You can also create custom rate limiters in App\Providers\RouteServiceProvider for more complex scenarios.
