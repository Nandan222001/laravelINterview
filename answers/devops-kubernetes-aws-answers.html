Quick Reference Answers (100 Questions)

1. What is Kubernetes and when should you use it?
Kubernetes (K8s) orchestrates containerized applications at scale. Manages deployment, scaling, networking, storage. Nodes run container runtime, control plane manages cluster. Ideal for microservices, high availability, horizontal scaling.
2. Explain Docker containerization basics.
Docker containerization packages application with dependencies in image. Build from Dockerfile, run containers from image. Lightweight VMs, fast startup, consistent environment across dev/staging/production.
3. What are multi-stage Docker builds?
Multi-stage Docker builds reduce image size. Compile/build in first stage, copy artifacts to final stage, avoid shipping build dependencies. Example: Alpine base, compile in builder, copy binary.
4. How do CI/CD pipelines work?
CI/CD pipelines automate build, test, deploy. GitHub Actions, Jenkins, GitLab CI execute on code push. Run tests, build artifacts, deploy to servers/containers. Faster feedback, fewer manual errors.
5. What are Jenkins declarative pipelines?
Jenkins declarative pipelines define build steps as code. Stages (build, test, deploy), agents (runners), tools (Docker, Maven), post-build actions, notifications.
6. Explain GitHub Actions workflows.
GitHub Actions workflows trigger on events (push, PR), run jobs in parallel, each job has steps (checkout, build, test, deploy). Use community actions, reuse workflows across repos.
7. What is Infrastructure as Code (IaC)?
Infrastructure as Code (IaC) defines infrastructure declaratively. Terraform, CloudFormation, Ansible describe resources, version control, repeat deployment. Benefits: reproducibility, disaster recovery, change tracking.
8. How does Terraform provisioning work?
Terraform provisioning: define providers (AWS, GCP, Azure), resources (instances, databases), state file tracks real infrastructure. `terraform plan` previews changes, `terraform apply` makes changes.
9. What is a service mesh?
Service mesh (Istio, Linkerd) manages service-to-service communication. Handles routing, retry, circuit breaking, mTLS encryption, observability without app code changes.
10. Explain Istio mTLS.
Istio mTLS: automatic mutual TLS between services. Transparently encrypts traffic, verifies identity via certificates. Zero-trust security at network level.

#!/usr/bin/env python3 import numpy as np from scipy import stats def monitor_process_control(): """Monitor processes using statistical process control""" metrics = collect_process_metrics() control_charts = {} for metric_name, values in metrics.items(): # Calculate control limits mean = np.mean(values) std_dev = np.std(values) ucl = mean + 3 * std_dev # Upper Control Limit lcl = mean - 3 * std_dev # Lower Control Limit # Detect violations violations = [] # Rule 1: Point beyond control limits for i, value in enumerate(values): if value > ucl or value < lcl: violations.append({ 'rule': 'Beyond control limits', 'index': i, 'value': value }) # Rule 2: 7 consecutive points on same side of mean consecutive = 0 for value in values: if value > mean: consecutive = consecutive + 1 if consecutive > 0 else 1 else: consecutive = consecutive - 1 if consecutive < 0 else -1 if abs(consecutive) >= 7: violations.append({ 'rule': '7 consecutive points trend', 'consecutive': consecutive }) chart = { 'mean': mean, 'std_dev': std_dev, 'ucl': ucl, 'lcl': lcl, 'violations': violations, 'in_control': len(violations) == 0 } # Create investigation ticket if out of control if not chart['in_control']: create_investigation_ticket({ 'metric': metric_name, 'violations': violations, 'chart_data': chart }) control_charts[metric_name] = chart update_spc_dashboard(control_charts) return control_charts
