1. CodeIgniter Architecture

Overview

CodeIgniter is a lightweight PHP framework that follows the Model-View-Controller (MVC) architectural pattern. It's known for its small footprint, exceptional performance, and minimal configuration requirements.

Key Components

Component Description Location (CI3)
Application Contains your application code application/
System CodeIgniter core files system/
Index.php Front controller/entry point index.php

Directory Structure (CI3)

codeigniter/
├── application/
│   ├── config/          # Configuration files
│   ├── controllers/     # Application controllers
│   ├── models/          # Data models
│   ├── views/           # View templates
│   ├── libraries/       # Custom libraries
│   ├── helpers/         # Helper functions
│   ├── hooks/           # Hook files
│   ├── language/        # Language files
│   ├── core/            # Extend core classes
│   └── cache/           # Cache files
├── system/              # CI core framework
├── user_guide/          # Documentation
└── index.php            # Entry point

Directory Structure (CI4)

codeigniter4/
├── app/
│   ├── Config/          # Configuration files
│   ├── Controllers/     # Application controllers
│   ├── Models/          # Data models
│   ├── Views/           # View templates
│   ├── Filters/         # HTTP filters (replaces hooks)
│   ├── Libraries/       # Custom libraries
│   ├── Helpers/         # Helper functions
│   └── Database/        # Migrations & seeds
├── public/              # Web root (index.php, assets)
├── system/              # CI4 core framework
├── writable/            # Cache, logs, uploads
└── vendor/              # Composer dependencies

Application Flow

1. index.php (Entry Point)
   ↓
2. CodeIgniter.php (Bootstrap)
   ↓
3. Router (Route Matching)
   ↓
4. Hooks (pre_controller)
   ↓
5. Controller (Load & Execute)
   ↓
6. Model (Database Operations)
   ↓
7. View (Render Template)
   ↓
8. Hooks (post_controller)
   ↓
9. Output (Send to Browser)
💡 Key Principle: CodeIgniter follows a "nearly zero configuration" philosophy, allowing developers to start building immediately with sensible defaults.

2. MVC Pattern in CodeIgniter

Understanding MVC

MVC separates application logic into three interconnected components, promoting organized code and separation of concerns.

Layer Responsibility Example
Model Data logic, database operations User_model.php
View Presentation layer, HTML output user_list.php
Controller Request handling, business logic Users.php

MVC Flow Example

<?php
// Controller: application/controllers/Users.php (CI3)
class Users extends CI_Controller {
    
    public function __construct() {
        parent::__construct();
        $this->load->model('user_model');
    }
    
    public function index() {
        // Get data from model
        $data['users'] = $this->user_model->get_all_users();
        
        // Load view with data
        $this->load->view('users/index', $data);
    }
    
    public function show($id) {
        $data['user'] = $this->user_model->get_user($id);
        
        if (empty($data['user'])) {
            show_404();
        }
        
        $this->load->view('users/show', $data);
    }
}

// Model: application/models/User_model.php (CI3)
class User_model extends CI_Model {
    
    public function get_all_users() {
        return $this->db->get('users')->result();
    }
    
    public function get_user($id) {
        return $this->db->get_where('users', ['id' => $id])->row();
    }
    
    public function create_user($data) {
        return $this->db->insert('users', $data);
    }
    
    public function update_user($id, $data) {
        $this->db->where('id', $id);
        return $this->db->update('users', $data);
    }
    
    public function delete_user($id) {
        return $this->db->delete('users', ['id' => $id]);
    }
}
✅ Best Practice: Keep controllers thin by moving complex business logic to models or libraries. Controllers should orchestrate, not implement.

3. Routing

URI Routing Basics

CodeIgniter uses URI segments to determine which controller and method to call. The default pattern is:

example.com/controller/method/param1/param2/param3

Route Configuration (CI3)

<?php
// application/config/routes.php

// Default controller
$route['default_controller'] = 'welcome';
$route['404_override'] = '';
$route['translate_uri_dashes'] = FALSE;

// Basic routing
$route['about'] = 'pages/about';
$route['contact'] = 'pages/contact';

// Wildcards (:num and :any)
$route['product/(:num)'] = 'catalog/product_lookup/$1';
$route['blog/(:any)'] = 'blog/post/$1';

// Regular expressions
$route['products/([a-z]+)/(:num)'] = 'catalog/product_lookup_by_name/$1/$2';

// Multiple parameters
$route['user/(:num)/post/(:num)'] = 'posts/show/$1/$2';

// HTTP verb routing (CI3.x+)
$route['api/users']['GET'] = 'api/users/index';
$route['api/users']['POST'] = 'api/users/create';
$route['api/users/(:num)']['GET'] = 'api/users/show/$1';
$route['api/users/(:num)']['PUT'] = 'api/users/update/$1';
$route['api/users/(:num)']['DELETE'] = 'api/users/delete/$1';

Route Configuration (CI4)

<?php
// app/Config/Routes.php

namespace Config;

$routes = Services::routes();

$routes->setDefaultNamespace('App\\Controllers');
$routes->setDefaultController('Home');
$routes->setDefaultMethod('index');

// Basic routes
$routes->get('/', 'Home::index');
$routes->get('about', 'Pages::about');

// HTTP verb routes
$routes->get('users', 'Users::index');
$routes->get('users/(:num)', 'Users::show/$1');
$routes->post('users', 'Users::create');
$routes->put('users/(:num)', 'Users::update/$1');
$routes->delete('users/(:num)', 'Users::delete/$1');

// Route groups
$routes->group('admin', ['filter' => 'auth'], function($routes) {
    $routes->get('dashboard', 'Admin\\Dashboard::index');
    $routes->get('users', 'Admin\\Users::index');
});

// Named routes
$routes->get('profile', 'User::profile', ['as' => 'user_profile']);
💡 Pro Tip: In CI4, routes are processed in the order they are defined. Place more specific routes before general ones.

4. Controllers

Basic Controller (CI3)

<?php
class Blog extends CI_Controller {
    
    public function __construct() {
        parent::__construct();
        $this->load->model('blog_model');
        $this->load->helper(['url', 'form']);
    }
    
    public function index() {
        $data['posts'] = $this->blog_model->get_posts();
        $data['title'] = 'Blog Posts';
        
        $this->load->view('templates/header', $data);
        $this->load->view('blog/index', $data);
        $this->load->view('templates/footer');
    }
    
    public function view($slug = NULL) {
        $data['post'] = $this->blog_model->get_posts($slug);
        
        if (empty($data['post'])) {
            show_404();
        }
        
        $data['title'] = $data['post']['title'];
        $this->load->view('blog/view', $data);
    }
    
    public function create() {
        $this->load->library('form_validation');
        
        $this->form_validation->set_rules('title', 'Title', 'required');
        $this->form_validation->set_rules('text', 'Text', 'required');
        
        if ($this->form_validation->run() === FALSE) {
            $this->load->view('blog/create');
        } else {
            $this->blog_model->set_post();
            redirect('blog');
        }
    }
}

Basic Controller (CI4)

<?php
namespace App\\Controllers;

use App\\Models\\BlogModel;

class Blog extends BaseController {
    
    protected $blogModel;
    
    public function __construct() {
        $this->blogModel = new BlogModel();
        helper(['url', 'form']);
    }
    
    public function index() {
        $data = [
            'posts' => $this->blogModel->findAll(),
            'title' => 'Blog Posts'
        ];
        
        return view('templates/header', $data)
             . view('blog/index', $data)
             . view('templates/footer');
    }
    
    public function show($slug = null) {
        $post = $this->blogModel->where('slug', $slug)->first();
        
        if (!$post) {
            throw \\CodeIgniter\\Exceptions\\PageNotFoundException::forPageNotFound();
        }
        
        return view('blog/show', ['post' => $post]);
    }
    
    public function create() {
        if (!$this->validate([
            'title' => 'required|min_length[3]',
            'text'  => 'required'
        ])) {
            return view('blog/create', ['validation' => $this->validator]);
        }
        
        $this->blogModel->save($this->request->getPost());
        return redirect()->to('/blog');
    }
}
⚠️ Security: Always validate and sanitize user input in controllers. Use form validation library to prevent SQL injection and XSS attacks.

5. Models

Basic Model (CI3)

<?php
class User_model extends CI_Model {
    
    public function __construct() {
        parent::__construct();
        $this->load->database();
    }
    
    public function get_users($limit = NULL) {
        if ($limit) {
            $this->db->limit($limit);
        }
        $query = $this->db->get('users');
        return $query->result();
    }
    
    public function get_user($id) {
        $query = $this->db->get_where('users', ['id' => $id]);
        return $query->row();
    }
    
    public function create_user($data) {
        return $this->db->insert('users', $data);
    }
    
    public function update_user($id, $data) {
        $this->db->where('id', $id);
        return $this->db->update('users', $data);
    }
    
    public function delete_user($id) {
        return $this->db->delete('users', ['id' => $id]);
    }
}

Model with Query Builder (CI4)

<?php
namespace App\\Models;

use CodeIgniter\\Model;

class UserModel extends Model {
    
    protected $table      = 'users';
    protected $primaryKey = 'id';
    protected $returnType = 'array';
    protected $useSoftDeletes = true;
    
    protected $allowedFields = ['name', 'email', 'password', 'status'];
    
    protected $useTimestamps = true;
    protected $createdField  = 'created_at';
    protected $updatedField  = 'updated_at';
    protected $deletedField  = 'deleted_at';
    
    protected $validationRules = [
        'name'  => 'required|min_length[3]',
        'email' => 'required|valid_email|is_unique[users.email]'
    ];
    
    public function getUserWithPosts($id) {
        return $this->select('users.*, COUNT(posts.id) as post_count')
                    ->join('posts', 'posts.user_id = users.id', 'left')
                    ->where('users.id', $id)
                    ->groupBy('users.id')
                    ->first();
    }
}
✅ Best Practice: Keep models focused on database operations. Complex business logic should be in libraries or services.

7. Helpers vs Libraries

Key Differences

Aspect Helpers Libraries
Structure Procedural functions Object-oriented classes
Loading $this->load->helper('name') $this->load->library('name')
Usage Direct function calls Class methods via $this->name
State Stateless Can maintain state

Creating Custom Helper

<?php
// application/helpers/custom_helper.php

if (!function_exists('format_phone')) {
    function format_phone($phone) {
        $phone = preg_replace('/[^0-9]/', '', $phone);
        if (strlen($phone) == 10) {
            return preg_replace('/([0-9]{3})([0-9]{3})([0-9]{4})/', '($1) $2-$3', $phone);
        }
        return $phone;
    }
}

if (!function_exists('time_ago')) {
    function time_ago($timestamp) {
        $time = time() - strtotime($timestamp);
        if ($time < 60) return 'just now';
        if ($time < 3600) return floor($time / 60) . ' minutes ago';
        return floor($time / 3600) . ' hours ago';
    }
}

8. Hooks

What are Hooks?

Hooks allow you to tap into and modify the framework's inner workings without hacking core files. They execute custom scripts at various points in the execution process.

Hook Points

Hook Point When It Executes
pre_system Very early, before system execution
pre_controller Before controller is called
post_controller_constructor After controller instantiation
post_controller After controller execution
post_system After final page sent to browser

Enabling and Defining Hooks (CI3)

<?php
// application/config/config.php
$config['enable_hooks'] = TRUE;

// application/config/hooks.php
$hook['post_controller'] = array(
    'class'    => 'Logger',
    'function' => 'log_activity',
    'filename' => 'Logger.php',
    'filepath' => 'hooks'
);

Hook Class Example

<?php
// application/hooks/Logger.php

class Logger {
    public function log_activity() {
        $CI =& get_instance();
        
        $data = array(
            'uri'       => $CI->uri->uri_string(),
            'method'    => $CI->input->method(),
            'ip'        => $CI->input->ip_address(),
            'timestamp' => date('Y-m-d H:i:s')
        );
        
        $CI->db->insert('activity_log', $data);
    }
}

Filters in CI4 (Replaces Hooks)

<?php
namespace App\\Filters;

use CodeIgniter\\Filters\\FilterInterface;

class AuthFilter implements FilterInterface {
    
    public function before($request, $arguments = null) {
        if (!session()->get('logged_in')) {
            return redirect()->to('/login');
        }
    }
    
    public function after($request, $response, $arguments = null) {
        // After controller execution
    }
}
⚠️ Important: CI4 uses Filters instead of Hooks. Filters provide better organization and more flexibility.

9. Database Query Builder

SELECT Queries

// Get all records
$query = $this->db->get('users');

$this->db->select('name, email');

$query = $this->db->get();

// Where clauses
$this->db->where('status', 'active');
$this->db->where('age >', 18);
$this->db->where_in('id', [1, 2, 3]);

$this->db->like('name', 'john');

$this->db->select('users.*, profiles.bio');
$this->db->from('users');
$this->db->join('profiles', 'profiles.user_id = users.id', 'left');

// ORDER BY and LIMIT
$this->db->order_by('created_at', 'DESC');
$this->db->limit(10);

<?php

$data = array(
    'name' => 'John Doe',

$this->session->set_userdata('username', 'johndoe');

    'username' => 'johndoe',

11. Form Validation

Form Validation (CI3)

<?php
// Load validation library
$this->load->library('form_validation');

// Set validation rules
$this->form_validation->set_rules('username', 'Username', 'required|min_length[5]|max_length[12]');
$this->form_validation->set_rules('email', 'Email', 'required|valid_email|is_unique[users.email]');
$this->form_validation->set_rules('password', 'Password', 'required|min_length[8]');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required|matches[password]');

// Run validation
if ($this->form_validation->run() == FALSE) {
    // Validation failed
    $this->load->view('signup_form');
} else {
    // Validation passed
    $this->load->view('success');
}

Validation Rules

Rule Description
required Field must contain a value
min_length[n] Minimum length
max_length[n] Maximum length
valid_email Valid email address
alpha Alphabetic characters only
numeric Numeric characters only
is_unique[table.field] Value is unique in database
matches[field] Matches another field

Form Validation (CI4)

<?php
// In controller
if (!$this->validate([
    'username' => 'required|min_length[5]|max_length[12]',
    'email'    => 'required|valid_email|is_unique[users.email]',
    'password' => 'required|min_length[8]'
])) {
    return view('signup_form', ['validation' => $this->validator]);
}

// In view, display errors
<?= \\Config\\Services::validation()->listErrors() ?>
✅ Best Practice: Always validate on server-side even if you have client-side validation. Never trust user input.

12. File Upload

File Upload (CI3)

<?php
// Controller
public function do_upload() {
    $config['upload_path']   = './uploads/';
    $config['allowed_types'] = 'gif|jpg|png|jpeg';
    $config['max_size']      = 2048; // 2MB
    $config['max_width']     = 1920;
    $config['max_height']    = 1080;
    $config['encrypt_name']  = TRUE; // Randomize filename
    
    $this->load->library('upload', $config);
    
    if (!$this->upload->do_upload('userfile')) {
        $error = $this->upload->display_errors();
        $this->load->view('upload_form', ['error' => $error]);
    } else {
        $data = $this->upload->data();
        $this->load->view('upload_success', $data);
    }
}

File Upload (CI4)

<?php
// Controller
public function upload() {
    $validation = \\Config\\Services::validation();
    
    $validation->setRules([
        'userfile' => [
            'label' => 'Image File',
            'rules' => 'uploaded[userfile]|max_size[userfile,2048]|is_image[userfile]'
        ]
    ]);
    
    if (!$validation->run()) {
        return view('upload_form', ['errors' => $validation->getErrors()]);
    }
    
    $file = $this->request->getFile('userfile');
    
    if ($file->isValid() && !$file->hasMoved()) {
        $newName = $file->getRandomName();
        $file->move('./uploads/', $newName);
        
        return view('upload_success', ['file' => $newName]);
    }
}
⚠️ Security: Always validate file types and sizes. Never trust user-supplied filenames. Store uploads outside web root when possible.

13. Migration from CI2 to CI3

Major Changes

Change CI2 CI3
File Names lowercase.php Ucfirst.php (PSR-4)
Class Names Lowercase Ucfirst
Database Driver mysql mysqli (deprecated mysql)
Security xss_clean() Deprecated, use htmlspecialchars()

Step-by-Step Migration

  1. Backup everything: Database and application files
  2. Check PHP version: CI3 requires PHP 5.3.7+
  3. Update database driver: Change 'mysql' to 'mysqli'
  4. Rename files: Controllers, models, libraries to Ucfirst
  5. Update class names: Match file names
  6. Update database configuration: New structure in database.php
  7. Replace deprecated functions:

Deprecated Functions

<?php
// CI2: XSS Clean
$clean = $this->security->xss_clean($data);

// CI3: Use PHP's htmlspecialchars instead
$clean = htmlspecialchars($data, ENT_QUOTES, 'UTF-8');

// CI2: SHA1
$hash = sha1($password);

// CI3: Use password_hash
$hash = password_hash($password, PASSWORD_DEFAULT);
💡 Migration Tip: Test thoroughly after migration. Focus on database queries, session handling, and security functions.

14. Migration from CI3 to CI4

Major Architectural Changes

Aspect CI3 CI4
PHP Version 5.3.7+ 7.4+ (8.0+ recommended)
Directory Structure application/ app/
Namespace Not required Required (PSR-4)
Autoloading Manual loading Composer autoloader
Models Basic Feature-rich with validation
Hooks Hooks system Filters (before/after)

Step-by-Step Migration Strategy

  1. Fresh Install: Install CI4 in a new directory
  2. Update PHP: Ensure PHP 7.4+ is available
  3. Migrate Configuration: Port settings to new config files
  4. Add Namespaces: Add namespace declarations to all classes
  5. Migrate Controllers: Update to use CI4 syntax
  6. Migrate Models: Extend CodeIgniter\\Model
  7. Update Routes: Use new routing syntax
  8. Migrate Views: Minimal changes required
  9. Convert Hooks to Filters: Rewrite hook logic
  10. Update Database Queries: Use new query builder syntax
  11. Test Thoroughly: Test all functionality

// CI3: application/controllers/Users.php

Key Differences Summary

  • Namespaces: Required in CI4, optional in CI3
  • Autoloading: Composer-based in CI4
  • View Rendering: return view() instead of $this->load->view()
  • Models: Much more powerful with built-in features
  • Request/Response: Proper HTTP message interfaces
  • Filters: Replace hooks with better architecture
  • Environment: Better .env file support
  • CLI: Improved command-line tools
⚠️ Important: CI4 is a complete rewrite, not a drop-in replacement. Plan for significant refactoring time and thorough testing.
✅ Best Practice: Migrate incrementally. Start with models and controllers, test thoroughly, then move to more complex features. Consider keeping CI3 and CI4 running in parallel during migration.
