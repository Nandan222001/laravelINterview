1WebSockets

WebSockets maintain a persistent, bidirectional connection between client and server. Unlike traditional HTTP which follows a request/response pattern, WebSocket allows the server to push data to the client at any time without requiring a client request.

How WebSockets Work

WebSockets are enabled with the ws:// protocol (or wss:// for secure connections). The connection starts with an HTTP upgrade request, then switches to the WebSocket protocol.

Key Benefits:
  • Full-duplex communication (both directions simultaneously)
  • Lower latency compared to HTTP polling
  • Reduced overhead after connection establishment
  • Real-time data streaming capabilities

WebSocket Connection Lifecycle

// Client-side WebSocket connection
const ws = new WebSocket('ws://localhost:8080');

ws.onopen = (event) => {
    console.log('Connection established');
    ws.send('Hello Server!');
};

ws.onmessage = (event) => {
    console.log('Message from server:', event.data);
};

ws.onerror = (error) => {
    console.error('WebSocket error:', error);
};

ws.onclose = (event) => {
    console.log('Connection closed:', event.code, event.reason);
};

Server-Side WebSocket (Node.js Example)

const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 8080 });

wss.on('connection', (ws) => {
    console.log('Client connected');

    ws.on('message', (message) => {
        console.log('Received:', message);
        // Broadcast to all clients
        wss.clients.forEach((client) => {
            if (client.readyState === WebSocket.OPEN) {
                client.send(message);
            }
        });
    });

    ws.on('close', () => {
        console.log('Client disconnected');
    });
});

WebSocket Graceful Shutdown

// Graceful shutdown process
function gracefulShutdown() {
    // 1. Stop accepting new connections
    wss.close(() => {
        console.log('No longer accepting new connections');
    });

    // 2. Send close frames to existing connections
    wss.clients.forEach((client) => {
        client.close(1000, 'Server shutting down');
    });

    // 3. Wait for close acknowledgments
    setTimeout(() => {
        // 4. Force close after timeout
        wss.clients.forEach((client) => {
            if (client.readyState !== WebSocket.CLOSED) {
                client.terminate();
            }
        });
        
        // 5. Shutdown server
        process.exit(0);
    }, 10000);
}

process.on('SIGTERM', gracefulShutdown);
process.on('SIGINT', gracefulShutdown);

2Server-Sent Events (SSE)

Server-Sent Events provide unidirectional communication from server to client. The browser opens a persistent HTTP connection, and the server sends events using a simple text-based format with data: lines.

SSE vs WebSocket

Feature SSE WebSocket
Direction Server to Client only Bidirectional
Protocol HTTP WebSocket protocol
Complexity Simpler More complex
Auto-reconnect Built-in Manual implementation
Data Format Text only Text and Binary

SSE Implementation

// Server-side (Node.js/Express)
app.get('/events', (req, res) => {
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-cache');
    res.setHeader('Connection', 'keep-alive');

    // Send event every 2 seconds
    const intervalId = setInterval(() => {
        const data = {
            time: new Date().toISOString(),
            message: 'Server update'
        };
        
        res.write(`data: ${JSON.stringify(data)}\n\n`);
    }, 2000);

    req.on('close', () => {
        clearInterval(intervalId);
        res.end();
    });
});

// Client-side
const eventSource = new EventSource('/events');

eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log('Received:', data);
};

eventSource.onerror = (error) => {
    console.error('SSE error:', error);
    eventSource.close();
};

Named Events with SSE

// Server: sending named events
res.write(`event: userJoined\n`);

3Laravel Broadcasting

Laravel Broadcasting provides an abstraction layer for real-time event broadcasting. It supports multiple drivers including Pusher (third-party service), Redis (self-hosted), and Log (for testing).

Laravel Broadcasting Drivers

  • Pusher: Third-party managed service, easy to set up, requires subscription
  • Redis: Self-hosted solution, requires Redis server and Socket.io/Laravel Echo Server
  • Log: For development and testing, writes events to log files
  • Null: Disables broadcasting entirely

Creating a Broadcastable Event

// app/Events/OrderShipped.php
namespace App\Events;

use Illuminate\Broadcasting\Channel;
use Illuminate\Broadcasting\InteractsWithSockets;
use Illuminate\Contracts\Broadcasting\ShouldBroadcast;
use Illuminate\Queue\SerializesModels;

class OrderShipped implements ShouldBroadcast
{
    use InteractsWithSockets, SerializesModels;

    public $order;

    public function __construct($order)
    {
        $this->order = $order;
    }

    public function broadcastOn()
    {
        return new Channel('orders');
    }

    public function broadcastWith()
    {
        return [
            'id' => $this->order->id,
            'total' => $this->order->total,
            'status' => 'shipped'
        ];
    }

    public function broadcastAs()
    {
        return 'order.shipped';
    }
}

4Redis Pub/Sub

Redis pub/sub allows you to publish messages to channels where subscribers receive them instantly. This is particularly useful for horizontal scaling, as multiple servers can subscribe to the same Redis instance to share events.

Redis Pub/Sub Architecture

Redis provides a publish/subscribe messaging paradigm where:

  • Publishers send messages to channels
  • Subscribers listen to one or more channels
  • Messages are delivered to all active subscribers
  • No message persistence (fire-and-forget)

Basic Redis Pub/Sub Implementation

// Publisher (Node.js)
const redis = require('redis');
const publisher = redis.createClient();

// Publish message to channel
publisher.publish('notifications', JSON.stringify({
    type: 'order.created',
    data: { orderId: 123, total: 99.99 }
}));

// Subscriber (Node.js)
const subscriber = redis.createClient();

subscriber.on('message', (channel, message) => {
    const data = JSON.parse(message);
    console.log(`Received from ${channel}:`, data);
    
    // Process the message
    handleNotification(data);
});

subscriber.subscribe('notifications');

// Subscribe to multiple channels
subscriber.subscribe('notifications', 'alerts', 'updates');

Pattern Subscription

// Subscribe to all channels matching a pattern
subscriber.psubscribe('user.*', (channel, message) => {
    // Matches: user.123, user.456, user.notifications, etc.
    console.log(`Pattern match from ${channel}:`, message);
});

// Publish to specific user channels
publisher.publish('user.123', JSON.stringify({ action: 'logout' }));
publisher.publish('user.456', JSON.stringify({ action: 'message' }));

Redis Pub/Sub for Scaling WebSocket

// WebSocket server using Redis for multi-server scaling
const WebSocket = require('ws');
const redis = require('redis');

const wss = new WebSocket.Server({ port: 8080 });
const subscriber = redis.createClient();
const publisher = redis.createClient();

// Subscribe to broadcast channel
subscriber.subscribe('websocket-broadcast');

// When receiving from Redis, send to all WebSocket clients
subscriber.on('message', (channel, message) => {
    wss.clients.forEach((client) => {
        if (client.readyState === WebSocket.OPEN) {
            client.send(message);
        }
    });
});

// When receiving from WebSocket, publish to Redis
wss.on('connection', (ws) => {
    ws.on('message', (message) => {
        // Publish to Redis so all servers receive it
        publisher.publish('websocket-broadcast', message);
    });
});
Redis Pub/Sub Limitations:
  • No message persistence - if subscriber is offline, messages are lost
  • No acknowledgment - fire-and-forget delivery
  • Consider Redis Streams for reliable message delivery with persistence

5Real-time Sync Strategies

Optimistic Updates

Update the UI immediately and sync with the server asynchronously. This provides the best user experience but requires conflict resolution.

// Optimistic update example
function updatePost(postId, newContent) {
    // 1. Update UI immediately
    updateUIOptimistically(postId, newContent);
    
    // 2. Send to server
    fetch(`/api/posts/${postId}`, {
        method: 'PUT',
        body: JSON.stringify({ content: newContent })
    })
    .then(response => response.json())
    .then(data => {
        // 3. Confirm or rollback
        if (data.success) {
            confirmUpdate(postId, data.post);
        } else {
            rollbackUpdate(postId);
        }
    })
    .catch(error => {
        // 4. Rollback on error
        rollbackUpdate(postId);
        showError('Update failed');
    });
}

Pessimistic Updates

Wait for server confirmation before updating the UI. More reliable but slower user experience.

// Pessimistic update example
function updatePost(postId, newContent) {
    // 1. Show loading state
    showLoadingState(postId);
    
    // 2. Send to server and wait
    fetch(`/api/posts/${postId}`, {
        method: 'PUT',
        body: JSON.stringify({ content: newContent })
    })
    .then(response => response.json())
    .then(data => {
        // 3. Update UI only after confirmation
        if (data.success) {
            updateUI(postId, data.post);
        } else {
            showError(data.message);
        }
        hideLoadingState(postId);
    })
    .catch(error => {
        showError('Update failed');
        hideLoadingState(postId);
    });
}

Eventual Consistency

Allow multiple sources of truth that synchronize over time. Useful for offline-first applications.

// Eventual consistency with conflict resolution
class SyncManager {
    constructor() {
        this.localDB = new LocalDatabase();
        this.pendingSync = [];
        this.lastSyncTimestamp = null;
    }

    // Save locally first
    async saveLocally(data) {
        data.localTimestamp = Date.now();
        data.syncStatus = 'pending';
        await this.localDB.save(data);
        this.pendingSync.push(data);
        this.scheduleSyncIfOnline();
    }

    // Sync when online
    async syncWithServer() {
        const pending = await this.localDB.getPending();
        
        for (const item of pending) {
            try {
                const serverData = await this.sendToServer(item);
                
                // Conflict resolution
                if (serverData.timestamp > item.localTimestamp) {
                    // Server wins - update local
                    await this.localDB.update(item.id, serverData);
                } else {
                    // Local wins - keep local version
                    await this.localDB.markSynced(item.id);
                }
            } catch (error) {
                console.error('Sync failed for item:', item.id);
            }
        }
        
        this.lastSyncTimestamp = Date.now();
    }

    // Operational Transform for collaborative editing
    applyOperationalTransform(localOps, serverOps) {
        // Transform operations to handle concurrent edits
        const transformedOps = [];
        
        serverOps.forEach(serverOp => {
            localOps.forEach(localOp => {
                if (conflictsWithOp(serverOp, localOp)) {
                    const resolved = resolveConflict(serverOp, localOp);
                    transformedOps.push(resolved);
                }
            });
        });
        
        return transformedOps;
    }
}

6Connection Management

Tracking Active Connections

// Connection tracking system
class ConnectionManager {
    constructor() {
        this.connections = new Map();
        this.userConnections = new Map();
    }

    addConnection(socketId, userId, metadata = {}) {
        const connection = {
            socketId,
            userId,
            connectedAt: Date.now(),
            lastActivity: Date.now(),
            metadata
        };
        
        this.connections.set(socketId, connection);
        
        // Track user's multiple connections
        if (!this.userConnections.has(userId)) {
            this.userConnections.set(userId, new Set());
        }
        this.userConnections.get(userId).add(socketId);
    }

    updateActivity(socketId) {
        const connection = this.connections.get(socketId);
        if (connection) {
            connection.lastActivity = Date.now();
        }
    }

    removeConnection(socketId) {
        const connection = this.connections.get(socketId);
        if (connection) {
            const userId = connection.userId;
            this.userConnections.get(userId)?.delete(socketId);
            
            // Remove user entry if no more connections
            if (this.userConnections.get(userId)?.size === 0) {
                this.userConnections.delete(userId);
            }
        }
        this.connections.delete(socketId);
    }

    getUserConnections(userId) {
        return Array.from(this.userConnections.get(userId) || []);
    }

    getActiveConnectionsCount() {
        return this.connections.size;
    }
}

Reconnection with Exponential Backoff

// Robust reconnection logic
class ReconnectingWebSocket {
    constructor(url) {
        this.url = url;
        this.reconnectAttempts = 0;
        this.maxReconnectAttempts = 10;
        this.baseDelay = 1000;
        this.maxDelay = 30000;
        this.reconnectTimer = null;
        
        this.connect();
    }

    connect() {
        this.ws = new WebSocket(this.url);
        
        this.ws.onopen = () => {
            console.log('Connected successfully');
            this.reconnectAttempts = 0;
            this.onConnectionEstablished();
        };
        
        this.ws.onclose = () => {
            this.scheduleReconnect();
        };
        
        this.ws.onerror = (error) => {
            console.error('WebSocket error:', error);
        };
    }

    scheduleReconnect() {
        if (this.reconnectAttempts >= this.maxReconnectAttempts) {
            console.error('Max reconnection attempts reached');
            this.onMaxRetriesExceeded();
            return;
        }
        
        const delay = Math.min(
            this.baseDelay * Math.pow(2, this.reconnectAttempts),
            this.maxDelay
        );
        
        // Add jitter to prevent thundering herd
        const jitter = Math.random() * 1000;
        const reconnectDelay = delay + jitter;
        
        console.log(`Reconnecting in ${reconnectDelay}ms (attempt ${this.reconnectAttempts + 1})`);
        
        this.reconnectTimer = setTimeout(() => {
            this.reconnectAttempts++;
            this.connect();
        }, reconnectDelay);
    }

    disconnect() {
        clearTimeout(this.reconnectTimer);
        if (this.ws) {
            this.ws.close();
        }
    }
}

7WebSocket Security

Authentication Before Connection

// Server-side authentication
const jwt = require('jsonwebtoken');

wss.on('connection', (ws, req) => {
    // Extract token from query string or headers
    const token = new URL(req.url, 'ws://localhost').searchParams.get('token');
    
    try {
        // Verify JWT token
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        ws.userId = decoded.userId;
        ws.authenticated = true;
        
        console.log(`Authenticated user ${ws.userId} connected`);
        
    } catch (error) {
        console.error('Authentication failed:', error);
        ws.close(1008, 'Authentication failed');
        return;
    }
    
    // Only authenticated connections proceed
    ws.on('message', (message) => {
        if (!ws.authenticated) {
            ws.close(1008, 'Not authenticated');
            return;
        }
        handleMessage(ws, message);
    });
});

Rate Limiting

// Rate limiting implementation
class RateLimiter {
    constructor(maxRequests = 100, windowMs = 60000) {
        this.maxRequests = maxRequests;
        this.windowMs = windowMs;
        this.requests = new Map();
    }

    isAllowed(userId) {
        const now = Date.now();
        const userRequests = this.requests.get(userId) || {
            count: 0,
            resetTime: now + this.windowMs
        };

        if (now > userRequests.resetTime) {
            userRequests.count = 0;
            userRequests.resetTime = now + this.windowMs;
        }

        userRequests.count++;
        this.requests.set(userId, userRequests);

        return userRequests.count <= this.maxRequests;
    }

    getRemainingRequests(userId) {
        const userRequests = this.requests.get(userId);
        if (!userRequests) return this.maxRequests;
        return Math.max(0, this.maxRequests - userRequests.count);
    }
}

const rateLimiter = new RateLimiter(50, 60000);

ws.on('message', (message) => {
    if (!rateLimiter.isAllowed(ws.userId)) {
        ws.send(JSON.stringify({
            error: 'Rate limit exceeded',
            remaining: 0,
            resetIn: rateLimiter.requests.get(ws.userId).resetTime - Date.now()
        }));
        return;
    }
    
    // Process message
    handleMessage(ws, message);
});

10Socket.IO

What is Socket.IO?

Socket.IO is a library that enables real-time, bidirectional communication between web clients and servers. It provides several features beyond raw WebSocket:

  • Automatic transport fallback: WebSocket → HTTP long-polling
  • Automatic reconnection: Built-in reconnection logic
  • Room and namespace support: Easy message segmentation
  • Acknowledgment callbacks: Confirm message delivery
  • Broadcasting helpers: Send to all, to room, except sender
  • Binary support: Efficient binary data transfer

        onlineUsers.delete(userId);
        io.emit('user:offline', {
            userId: userId
        });
    });
